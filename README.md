# Игра «Шарики»

## Что это за игра и как она работает

Это аркадная игра с физикой шариков на плоскости. На поле движутся разноцветные шарики; при столкновении они **объединяются в один** с **смешанным цветом** (субтрактивное смешивание, как краски). Шарики отскакивают от границ экрана и не вылетают за пределы окна.

### Основные механики

- **Всасывание (ЛКМ)** — по левому клику шарик под курсором «всасывается» и попадает в инвентарь.
- **Выплёвывание (ПКМ)** — по правому клику последний шарик из инвентаря возвращается на поле в позицию курсора.
- **Зона удаления** — красный прямоугольник в правом верхнем углу. Если центр шарика попадает в эту зону (например, вы «перетащили» его туда, выплюнув в зону), шарик удаляется с поля.

Количество шариков в инвентаре отображается внизу слева. В начале игры на поле появляется несколько десятков шариков со случайными цветами и скоростями.

### Структура проекта

- **logic.py** — игровая логика: движение, отражение от стен, объединение при касании, зона удаления, инвентарь (всасывание/выплёвывание). Без графики.
- **game_gui.py** — графический интерфейс на Tkinter: окно, канва, отрисовка шариков и зоны удаления, обработка кликов и игровой цикл.
- **gui.py** — точка входа: запуск игры одной командой.

## Запуск локально

### Зависимости

- **Python 3.7+** (рекомендуется 3.8 или новее).
- Модуль **tkinter** — обычно поставляется с Python. На Linux может потребоваться пакет `python3-tk`:

  ```bash
  # Ubuntu / Debian
  sudo apt install python3-tk
  ```

  Внешние pip-зависимости не требуются (всё на стандартной библиотеке).

### Запуск одной командой

Из корня проекта:

```bash
python gui.py
```

или, если интерпретатор вызывается как `python3`:

```bash
python3 gui.py
```

Альтернативно можно запустить напрямую графический модуль:

```bash
python game_gui.py
```

После запуска откроется окно с полем, подсказкой по управлению и красной зоной «УДАЛИТЬ» в углу.

## Сборка и запуск в Docker (графическое окно через X11)

Чтобы игра запускалась в графическом окне **внутри контейнера**, вывод отображается через **X11**: контейнер подключается к X-серверу хоста, и окно Tkinter отображается на вашем экране.

### Требования на хосте

- **Linux**: X-сервер (обычно уже запущен). Разрешите подключения от Docker:
  ```bash
  xhost +local:docker
  ```
- **macOS**: XQuartz. После установки запустите XQuartz, в настройках включите «Allow connections from network clients», затем:
  ```bash
  xhost +localhost
  ```
- **Windows**: нужен X-сервер (например VcXsrv, X410 или WSL2 с настроенным X). Запустите сервер и разрешите подключения; переменная `DISPLAY` должна указывать на этот сервер (например `host.docker.internal:0` при использовании VcXsrv).

### Сборка образа

В каталоге с проектом (где лежит `Dockerfile`):

```bash
docker build -t balls-game .
```

Имя образа `balls-game` можно заменить на любое другое.

### Запуск контейнера с графическим окном

Передайте в контейнер переменную `DISPLAY` и смонтируйте сокет X11 (на Linux):

```bash
docker run --rm -e DISPLAY=$DISPLAY -v /tmp/.X11-unix:/tmp/.X11-unix balls-game
```

Игра должна сразу запуститься, и окно появится на вашем экране.

- **macOS** (XQuartz): обычно `DISPLAY=host.docker.internal:0`, сокет может быть в `/tmp/.X11-unix` или через `-e DISPLAY=host.docker.internal:0` без volume.
- **Windows** (VcXsrv и т.п.): укажите свой DISPLAY, например:
  ```bash
  docker run --rm -e DISPLAY=host.docker.internal:0 balls-game
  ```

После закрытия окна игры контейнер завершится (`--rm` удалит контейнер).
